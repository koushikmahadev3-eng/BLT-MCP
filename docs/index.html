Three Powerful Layers

BLT-MCP exposes the full BLT ecosystem through three clean abstractions that any MCP-compatible client can use.

Resources blt://

Access structured BLT data through standardised blt:// URIs. Every entity in the system is a first-class resource.

  • blt://issues & blt://issues/{id}
  • blt://repos & blt://contributors
  • blt://leaderboards & blt://rewards
  • blt://workflows

Tools

Let AI agents take action on BLT — submit bugs, update statuses, award contributors, and add context to issues.

  • submit_issue — report bugs & vulnerabilities
  • award_bacon — gamification & points
  • update_issue_status — manage lifecycle
  • add_comment — add context to issues

Prompts

Guided AI workflows for common security tasks — from triage to remediation planning to contribution review.

  • triage_vulnerability — severity assessment
  • plan_remediation — fix planning
  • review_contribution — quality review

How It Works

BLT-MCP sits between your AI agent and the BLT API, translating natural language into structured security actions.

  1. 1

    Configure your MCP client

    Add BLT-MCP to your mcp-settings.json with your BLT API key. Works with Claude Desktop, Cline, and any MCP-compatible client.

  2. 2

    Talk to your AI agent naturally

    Ask your AI to "Submit a critical XSS vulnerability in the login form" or "Show me the leaderboard". No special syntax required.

  3. 3

    BLT-MCP translates and executes

    The server maps your intent to the right BLT resource or tool call, validates inputs, authenticates with the BLT API, and returns structured results.

  4. 4

    Your agent takes action

    The AI agent receives clean, structured data and can chain multiple calls — triage a bug, assign severity, notify contributors, and award bacon points in one workflow.

Quick Start

Up and running in under two minutes.

1 Install & Build

# Clone the repository

git clone https://github.com/OWASP-BLT/BLT-MCP.git

cd BLT-MCP


# Install dependencies

npm install


# Build the project

npm run build

2 Configure Environment

# Copy the example env file

cp .env.example .env


# Edit .env with your BLT API key

BLT_API_BASE=https://blt.owasp.org/api

BLT_API_KEY=your_api_key_here

3 Add to MCP Client

{

"mcpServers": {

"blt": {

"command": "node",

"args": ["/path/to/BLT-MCP/dist/index.js"],

"env": {

"BLT_API_KEY": "your_key"

}

}

}

}

4 Start Using It

# Example AI prompts (in your IDE/chat):


"Submit a critical XSS vulnerability

in the login form of repo 123"


"Show me the BLT leaderboard"


"Help me triage this SQL injection"

API Reference

A quick reference for the most commonly used tools and resources.

  • Tools
  • Resources
  • Prompts

Tools

submit_issue write

Report new bugs and vulnerabilities to BLT.

Parameter Type Description
title * string Issue title
description * string Detailed description
repo_id string Repository ID
severity enum low · medium · high · critical
type enum bug · vulnerability · feature · other
award_bacon write

Award bacon points to a contributor for their work.

Parameter Type Description
contributor_id * string Contributor ID
points * number Points to award
reason * string Reason for the award
update_issue_status write

Change the lifecycle status of a BLT issue.

Parameter Type Description
issue_id * string Issue ID
status * enum open · in_progress · resolved · closed · wont_fix
comment string Optional explanation

Resources

URI Returns
blt://issues JSON array of all issues
blt://issues/{id} JSON object — specific issue
blt://repos Tracked repositories
blt://contributors All contributors
blt://leaderboards Rankings & statistics
blt://rewards Rewards & bacon points
blt://workflows All workflows

Prompts

triage_vulnerability guided

Guides AI through vulnerability triage and severity assessment. Args: vulnerability_description* · affected_component

plan_remediation guided

Creates comprehensive remediation plans for security issues. Args: issue_id* · context

review_contribution guided

Evaluates contributions with quality assessment and bacon point recommendations. Args: contribution_id* · contribution_type
